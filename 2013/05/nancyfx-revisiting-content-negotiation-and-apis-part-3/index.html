NancyFX - Revisiting Content Negotiation & APIs (Part 3)

posted on 20 May 2013 | NancyFX
<ul> <li>Original Post: <a href="/2012/11/nancy-and-content-negotiation">NancyFX and Content Negotiation</a></li> <li><a href="/2013/04/nancyfx-revisiting-content-negotiation-and-apis-part-1/">NancyFX - Revisiting Content Negotiation &amp; APIs (Part 1)</a></li> <li><a href="/2013/05/nancyfx-revisiting-content-negotiation-and-apis-part-2/">NancyFX - Revisiting Content Negotiation &amp; APIs (Part 2)</a></li> <li>NancyFX - Revisiting Content Negotiation &amp; APIs (Part 3)</li> </ul> <p>In this last part I want to show you creating your own media type, so far we have gone over a really basic example of returning a response and letting the accept header handle different results, and then we went over how explicitly using <code>Negotiate</code> allows you to customize your response more.</p> <p>Creating your own Media Type may be something you never have to do, but it can help you solve some strange scenarios, for instance you could have a user request an invoice, rather than clutter your code with:</p> <pre><code>if (requestedInvoice.Type == "pdf") { // get a pdf invoice } else if (requestedInvoice.Type == "word") { } else if (...) </code></pre> <p>You get the picture, this is tedious problematic work, every time we want to supply a new invoice type we need to modify the existing request, what we want to do is just handle a new media type and process it if we know about it.</p> <!--excerpt--> <p><span class="note"><strong>Note:</strong> To keep this simple I'm going to show a simple CSV Processor, you can look at NancyFX code on how a ViewProcessor works if you want to hook up a View to a Word/PDF processor.</span></p> <h2>Codez is what I need!</h2> <p>There's 3 things we need to create</p> <ol> <li>CsvProcessor</li> <li>CsvSerializer</li> <li>CsvResponse</li> </ol> <p>The <code>CsvProcessor</code> takes the request, checks to see if it can handle the request, and if it can, it uses the <code>CsvSerializer</code> to process the request and return a <code>CsvResponse</code></p> <h2>Csv Response</h2> <p>This class is modeled off the Json Response, it takes a model and a serializer, sets all the correct headers for the response, serializes the response, and this is what is returned to the client.</p> <pre><code>public class CsvResponse&lt;TModel&gt; : Response { public CsvResponse(TModel model, ISerializer serializer) { if (serializer == null) { throw new InvalidOperationException("CSV Serializer not set"); } this.Contents = GetJsonContents(model, serializer); this.ContentType = "text/csv"; this.StatusCode = HttpStatusCode.OK; } private static Action&lt;Stream&gt; GetJsonContents(TModel model, ISerializer serializer) { return stream =&gt; serializer.Serialize("text/csv", model, stream); } } public class CsvResponse : CsvResponse&lt;object&gt; { public CsvResponse(object model, ISerializer serializer) : base(model, serializer) { } } </code></pre> <p>You don't REALLY need this class, I would say its a highly recommended optional. If you don't create this class it just means your processor needs to do all the work of setting up the response.</p> <h2>Csv Serializer</h2> <p>To handle serialization of the model to a CSV file I'm using <code>ServiceStack.Text</code> which comes with its own serializer! </p> <pre><code>public class CsvSerializer : ISerializer { public bool CanSerialize(string contentType) { return IsCsvType(contentType); } public IEnumerable&lt;string&gt; Extensions { get { yield return "csv"; } } public void Serialize&lt;TModel&gt;(string contentType, TModel model, Stream outputStream) { using (var writer = new StreamWriter(new UnclosableStreamWrapper(outputStream))) { ServiceStack.Text.CsvSerializer.SerializeToWriter(model, writer); } } private static bool IsCsvType(string contentType) { if (string.IsNullOrEmpty(contentType)) { return false; } var contentMimeType = contentType.Split(';')[0]; return contentMimeType.Equals("text/csv", StringComparison.InvariantCultureIgnoreCase) || contentMimeType.StartsWith("text/csv", StringComparison.InvariantCultureIgnoreCase); } } </code></pre> <p>The serializer needs to implement <code>ISerializer</code>, NancyFX will automatically pick up and register this class for you, so all you need to do is create it.</p> <p>The <code>CanSerialize</code> method is very important, in the <code>CsvProcessor</code> we will invoke all serializers until we find one that can handle serializing the content for the requested processor.</p> <h2>Csv Processor</h2> <p>Lastly we have the processor.</p> <pre><code>public class CsvProcessor : IResponseProcessor { private readonly ISerializer serializer; private static readonly IEnumerable&lt;Tuple&lt;string, MediaRange&gt;&gt; extensionMappings = new[] { new Tuple&lt;string, MediaRange&gt;("csv", MediaRange.FromString("text/csv")) }; public CsvProcessor(IEnumerable&lt;ISerializer&gt; serializers) { this.serializer = serializers.FirstOrDefault(x =&gt; x.CanSerialize("text/csv")); } public IEnumerable&lt;Tuple&lt;string, MediaRange&gt;&gt; ExtensionMappings { get { return extensionMappings; } } public ProcessorMatch CanProcess(MediaRange requestedMediaRange, dynamic model, NancyContext context) { if (requestedMediaRange.Matches("text/csv")) { return new ProcessorMatch { ModelResult = MatchResult.DontCare, RequestedContentTypeResult = MatchResult.ExactMatch }; } return new ProcessorMatch { ModelResult = MatchResult.DontCare, RequestedContentTypeResult = MatchResult.NoMatch }; } public Response Process(MediaRange requestedMediaRange, dynamic model, NancyContext context) { return new CsvResponse(model, this.serializer); } } </code></pre> <p>This class has 3 methods and a constructor.</p> <p>The <code>Constructor</code> simply works out which serializer it requires.</p> <p><code>ExtensionMapping</code> specifies which extension can be used on a request to call the processor. i.e if you can't pass in accepts headers, you can end the url with <code>.csv</code> and the processor will be invoked!</p> <p><code>CanProcess</code> checks to see if the request can be processed.</p> <p>Lastly <code>Process</code> simply processes the request.</p> <h2>Lets see it in action!</h2> <p>Using the EXACT same routes we defined on the previous posts, we can simply modify the header to ask for <code>text/csv</code></p> <p><img src="/images/nancyfx-conneg-updated-part3-1.png" alt="" /></p> <p>And we get all our data serialized to CSV format! We didn't have to modify the bootstrapper or anything, Nancy just sees the implemented interfaces and says MINE! :D</p> <h2>What about the Extension mentioned earlier!</h2> <p>In the processor we had the following</p> <pre><code>private static readonly IEnumerable&lt;Tuple&lt;string, MediaRange&gt;&gt; extensionMappings = new[] { new Tuple&lt;string, MediaRange&gt;("csv", MediaRange.FromString("text/csv")) }; public IEnumerable&lt;Tuple&lt;string, MediaRange&gt;&gt; ExtensionMappings { get { return extensionMappings; } } </code></pre> <p>Well when you access a url with <code>.json</code> or <code>.csv</code>, it will response with that as a file!</p> <p>If we have the following page</p> <p><img src="/images/nancyfx-conneg-updated-part3-2.png" alt="" /></p> <p>We add the extension <code>.json</code> to the end</p> <p><img src="/images/nancyfx-conneg-updated-part3-3.png" alt="" /></p> <p>Nancy Replies! (The JSON format shown done using <a href="https://chrome.google.com/webstore/detail/jsonview/chklaanhfefbnpoihckbnefhakgolnmc?hl=en">JSONView chrome extension</a></p> <p>If we update to <code>.csv</code></p> <p><img src="/images/nancyfx-conneg-updated-part3-4.png" alt="" /></p> <p>And if we open the file up</p> <p><img src="/images/nancyfx-conneg-updated-part3-5.png" alt="" /></p> <p>That's all there is to it!</p> <p>Hope this helps anyone with Conneg or Conneg with NancyFX :)</p>
