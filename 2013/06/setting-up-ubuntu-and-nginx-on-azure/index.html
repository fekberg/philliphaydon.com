Setting up Ubuntu and nginx on Azure

posted on 26 Jun 2013 | Azure
<ul> <li>Part 1 - Setting up the Virtual Machine and nginx</li> <li><a href="/2013/06/setting-up-a-new-website-and-domain-on-nginx/">Part 2 - Setting up new Website and Domain</a></li> <li><a href="/2013/06/setting-up-mono-on-nginx/">Part 3 - Setting up Mono on nginx</a></li> <li><a href="/2013/07/setting-up-a-nancyfx-website/">Part 4 - Setting up a NancyFX website</a></li> <li><a href="/2013/07/setting-up-a-servicestack-service/">Part 5 - Setting up a ServiceStack web service</a></li> </ul> <p><span class="note"><strong>Note:</strong> I'm breaking this up into 5 parts because otherwise it gets far too long :(</span></p> <p>So after spending a week with OSX, I thought I would see how well I go with setting up Mono on Ubuntu... on Azure. </p> <p>Turns out it's actually rather simple!</p> <h2>Setting up the Virtual Machine</h2> <p>Setting up the virtual machine is pretty straight forward. From the Azure Portal, click the Virtual Machines tab, and select New. </p> <p>I create mine from the gallery rather than the quick option.</p> <p>Select <code>Ubuntu Server 13.04</code> from the options, give your new server a name. For Authentication I recommend making a password rather than uploading an SSH key. Only because if you're new or trying this out just to play around, that stuff is probably too difficult for now :)</p> <!--excerpt--> <p><img src="/images/setup-mono-on-ubuntu-1.png" alt="" /></p> <p>Click next (the little arrow thing) and setup as a Stand-Alone Virtual Machine. Name it and select a region, Next, and bam you're done.</p> <p><img src="/images/setup-mono-on-ubuntu-2.png" alt="" /></p> <p>Once the provisioning is complete, you need to click on the Virtual Machine, and select Endpoints. </p> <p><img src="/images/setup-mono-on-ubuntu-3.png" alt="" /></p> <p>Create a new endpoint for port 80.</p> <p><img src="/images/setup-mono-on-ubuntu-4.png" alt="" /></p> <p>This is so the websites are publicly accessible.</p> <h2>Logging into Ubuntu!</h2> <p>Unlike Windows Server, there's no desktop for us to RDP into, we could add it, but we wont... There's no need to :)</p> <p>First things first, we need to download PuTTY. So head on over to <a href="http://www.putty.org">http://www.putty.org</a> and <a href="http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html">download PuTTY</a>. </p> <p><img src="/images/setup-mono-on-ubuntu-5.png" alt="" /></p> <p>When you've downloaded it, run it up, and... switch back to the Azure Portal, to the Dashboard. On the <code>quick glance</code> section on the right hand side. Find the <code>SSH DETAILS</code> piece.</p> <p><img src="/images/setup-mono-on-ubuntu-6.png" alt="" /></p> <p>Enter the address into the PuTTY details:</p> <p><img src="/images/setup-mono-on-ubuntu-7.png" alt="" /></p> <p>And select <code>option</code>. You should get a console type window now that looks like.</p> <p><img src="/images/setup-mono-on-ubuntu-8.png" alt="" /></p> <p>Login using the details you created when creating the Virtual Machine, and if successful you should get a wall of text.</p> <blockquote> <p>login as: phillip phillip@mono-sample-ph.cloudapp.net's password:</p> <p>The programs included with the Ubuntu system are free software; the exact distribution terms for each program are described in the individual files in /usr/share/doc/*/copyright.</p> <p>Ubuntu comes with ABSOLUTELY NO WARRANTY, to the extent permitted by applicable law.</p> <p>Welcome to Ubuntu 13.04 (GNU/Linux 3.8.0-25-generic x86_64)</p> <ul> <li><p>Documentation: https://help.ubuntu.com/</p> <p>System information as of Wed Jun 26 14:46:18 UTC 2013</p> <p>System load: 0.02 Processes: 93 Usage of /: 3.1% of 28.79GB Users logged in: 0 Memory usage: 8% IP address for eth0: 10.78.154.86 Swap usage: 0%</p> <p>Graph this data and manage this system at https://landscape.canonical.com/</p> <p>Get cloud support with Ubuntu Advantage Cloud Guest: http://www.ubuntu.com/business/services/cloud</p> <p>Use Juju to deploy your cloud instances and workloads: https://juju.ubuntu.com/#cloud-raring</p></li> </ul> <p>0 packages can be updated. 0 updates are security updates.</p> <p>phillip@mono-sample-ph:~$</p> </blockquote> <p>Next, type in the command <code>sudo -s</code>, and enter your password again.</p> <p><img src="/images/setup-mono-on-ubuntu-9.png" alt="" /></p> <p>This will set the current user to a super user so we don't need to call <code>sudo</code> with every command, which can get a little frustrating after a while.</p> <h2>Setup nginx</h2> <p>Before we setup nginx, run the command <code>apt-get update</code>. This downloads all the package lists from the repositories and updates the information with the newest versions of the packages and their dependencies. </p> <p>Now, run the command <code>apt-get install nginx</code>.</p> <p><img src="/images/setup-mono-on-ubuntu-10.png" alt="" /></p> <p>Enter <code>y</code> and hit <kbd>enter</kbd>, and it will download the package and install.</p> <p>Next, run the command <code>service nginx restart</code>, this should say the service is restarting.</p> <pre><code>root@mono-sample-ph:~# service nginx restart * Restarting nginx nginx [ OK ] </code></pre> <p>Now, run the command <code>ls</code></p> <pre><code>root@mono-sample-ph:~# ls root@mono-sample-ph:~# </code></pre> <p>If you haven't change the directory then it should be currently empty. So now we can run the command <code>wget http://localhost</code> which will issue a request to the <code>localhost</code> domain and download the HTML file.</p> <p>You should end up with your command-line looking like:</p> <pre><code>root@mono-sample-ph:~# wget http://localhost --2013-06-26 15:07:43-- http://localhost/ Resolving localhost (localhost)... 127.0.0.1 Connecting to localhost (localhost)|127.0.0.1|:80... connected. HTTP request sent, awaiting response... 200 OK Length: 612 [text/html] Saving to: âindex.htmlâ 100%[======================================&gt;] 612 --.-K/s in 0s 2013-06-26 15:07:43 (56.7 MB/s) - âindex.htmlâ saved [612/612] root@mono-sample-ph:~# ls index.html root@mono-sample-ph:~# </code></pre> <p>As you can see when running <code>ls</code> again there's now a file called index.html. We can take a quick look at that by running <code>nano index.html</code> </p> <p>Nano is a text editor that is installed on Ubuntu by default. To exit out of Nano once you're done looking press <kbd>ctrl</kbd> + <kbd>x</kbd> which will close Nano and return you to the command-line.</p> <p>For a good beginners guide take a look at <a href="http://www.howtogeek.com/howto/42980/the-beginners-guide-to-nano-the-linux-command-line-text-editor/">The Beginner’s Guide to Nano, the Linux Command-Line Text Editor</a></p> <p><img src="/images/setup-mono-on-ubuntu-11.png" alt="" /></p> <p>And we can see that nginx is setup and running!</p>
