Setting up Mono on nginx

posted on 28 Jun 2013 | Azure
<ul> <li><a href="/2013/06/setting-up-ubuntu-and-nginx-on-azure/">Part 1 - Setting up the Virtual Machine and nginx</a></li> <li><a href="/2013/06/setting-up-a-new-website-and-domain-on-nginx">Part 2 - Setting up new Website and Domain on nginx</a></li> <li>Part 3 - Setting up Mono on nginx</li> <li><a href="/2013/07/setting-up-a-nancyfx-website/">Part 4 - Setting up a NancyFX website</a></li> <li><a href="/2013/07/setting-up-a-servicestack-service/">Part 5 - Setting up a ServiceStack web service</a></li> </ul> <p>Now we get to the fun stuff, getting up Mono. I'll be demoing with Mono 2.10 but after this series I may include a 6th part on how build Mono 3.0 from source and run it. </p> <h2>Setting up our test page</h2> <p>I figured we should start with setting up our test page. Something super simple, we will create a new page that outputs a bit of server side text. </p> <p>We wont deploy a bin folder or anything like that, it will be almost like deploying Classic ASP. </p> <pre><code>&lt;%@ Page Language="C#" AutoEventWireup="true" %&gt; &lt;!DOCTYPE html&gt; &lt;html xmlns="http://www.w3.org/1999/xhtml"&gt; &lt;head runat="server"&gt; &lt;title&gt;&lt;/title&gt; &lt;/head&gt; &lt;body&gt; &lt;div&gt; &lt;% var test = "Hello World! Got an aspx page running on nginx!!!"; %&gt; &lt;%= test %&gt; &lt;/div&gt; &lt;/body&gt; &lt;/html&gt; </code></pre> <p>So we are just creating a variable named <code>test</code> with some test data, then outputting it to the page. </p> <!--excerpt--> <p>We need to navigate to our site directory by calling <code>cd /var/www/phillip.pw</code> followed by creating a new file in nano by calling <code>nano test.aspx</code></p> <p>This will open up a new nano window. Now you can type in the test code, or paste it in. Now we need to save it by calling <kbd>CTRL</kbd> + <kbd>X</kbd> followed by <kbd>Y</kbd> and <kbd>Enter</kbd>. Same thing we have been doing in the other parts. </p> <h2>Installing Mono</h2> <p>Now that our test page is ready, we can install Mono by calling <code>apt-get install mono-complete</code>, your window should now look like</p> <p><img src="/images/setup-mono-on-ubuntu-part-3-1.png" alt="" /></p> <p>Hit <kbd>Y</kbd> and <kbd>Enter</kbd>, and let it install... this could take a while.</p> <p>Next we want to install the fastcgi-mono-server4 package. So call <code>apt-get install mono-fastcgi-server4</code> and install. This package is super small so will install quickly.</p> <p>To quickly test that Mono is installed call <code>mono -V</code> you should get something like so</p> <p><img src="/images/setup-mono-on-ubuntu-part-3-2.png" alt="" /></p> <h2>Configuring our website!</h2> <p>We need to head on over to the nginx server configuration. So call <code>cd /etc/nginx/sites-available</code> followed by <code>nano default</code> so we can edit our site to use the fastcgi stuff.</p> <p>We need to add the fastcgi stuff, its important to note the include path. If you remember from Part 1 I mentioned the <code>fastcgi_params</code> file, well we need to point to that file. We will modify that next.</p> <pre><code>server { # the port the site is listening on listen 80; # the hostname server_name *.phillip.pw; # the path to where we host the site root /var/www/phillip.pw; # the default page index index.html; location / { fastcgi_index /index.html; fastcgi_pass 127.0.0.1:9000; include /etc/nginx/fastcgi_params; } } </code></pre> <p><span class="note">Note how the fastcgi stuff is put into a location followed by a forward slash to indicate that its location is the root. If you put it in the root of server (with no location scope) you will end up with nginx throwing an error like</span> </p> <blockquote> <p>root@mono-sample-ph:/var/log/mono# nginx nginx: [emerg] "fastcgi_pass" directive is not allowed here in /etc/nginx/sites-enabled/default:34</p> </blockquote> <p>Save the file and exit. </p> <p>Now drop down 1 level by calling <code>cd ..</code> and then calling <code>ls</code> to view the directory listing.</p> <p><img src="/images/setup-mono-on-ubuntu-part-3-3.png" alt="" /></p> <p>We need to modify the fastcgi_params file, we open it up in nano... <code>nano fastcgi_params</code> and we need to add the two following lines of code to the bottom of the file:</p> <pre><code>fastcgi_param PATH_INFO ""; fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name; </code></pre> <p>So you should end up with something like</p> <p><img src="/images/setup-mono-on-ubuntu-part-3-4.png" alt="" /></p> <p>Save and exit that file. </p> <h2>Setting up the Mono Site</h2> <p>So we need to configure a site in Mono as well. This part was a little confusing for me initially. Basically we need to create all of this manually.</p> <p>Head back to <code>cd /etc</code> and call <code>ls</code>, in here you should see an file called <code>init.d</code>. Some tutorials on the internet show this file in <code>/etc/rc.d/init.d</code> but this is actually because some versions of Linux put it in 1 place, while other versions put it in the other. And I think those who put it in <code>rc.d</code> do a syslink to the root <code>init.d</code> version. </p> <p>In any-case on Ubuntu, <code>rc.d</code> does not exist. So we want to navigate into <code>init.d</code> so call <code>cd init.d</code></p> <p>Create a new directory in here called <code>mono-fastcgi</code> by calling <code>mkdir mono-fastcgi</code> and then navigate to it.</p> <p>Here we want to create a Web App file, this is a configuration file for our site.</p> <p>Create a new file with nano... <code>nano phillip.pw.webapp</code></p> <p>Write the following text into the file:</p> <pre><code>&lt;apps&gt; &lt;web-application&gt; &lt;name&gt;phillip.pw&lt;/name&gt; &lt;vhost&gt;*&lt;/vhost&gt; &lt;vport&gt;80&lt;/vport&gt; &lt;vpath&gt;/&lt;/vpath&gt; &lt;path&gt;/var/www/phillip.pw&lt;/path&gt; &lt;/web-application&gt; &lt;/apps&gt; </code></pre> <p>All these names and paths should be familiar if you've been following from Parts 1/2</p> <h2>Setting up the log files</h2> <p>Now we need to navigate to <code>cd /var/log</code> and create a new directory called mono <code>mkdir mono</code>, navigate into the directory, <code>cd mono</code>, and create a new file called <code>fastcgi.log</code>. To create a new file just call <code>&gt; fastcgi.log</code> which will create a new empty file.</p> <h2>Registering fastcgi-mono-server</h2> <p>Lastly we need to configure the server. We can start the FastCGI background process by calling</p> <p><code>fastcgi-mono-server4 --appconfigdir /etc/init.d/mono-fastcgi /socket=tcp:127.0.0.1:9000 /logfile=/var/log/mono/fastcgi.log &amp;</code></p> <p>(if it's hard to scroll, here's the same line broken down into pieces)</p> <pre><code>fastcgi-mono-server4 --appconfigdir /etc/init.d/mono-fastcgi /socket=tcp:127.0.0.1:9000 /logfile=/var/log/mono/fastcgi.log &amp; </code></pre> <p>Notice the paths match those we just created.</p> <p>You should have a terminal that looks like:</p> <p><img src="/images/setup-mono-on-ubuntu-part-3-5.png" alt="" /></p> <p>Now give nginx a restart by calling <code>service nginx restart</code></p> <h2>The big test</h2> <p><img src="/images/setup-mono-on-ubuntu-part-3-6.png" alt="" /></p> <p>BAM Just like that we now have an ASPX page running on Mono, on nginx, on Ubuntu, on Azure! </p>
