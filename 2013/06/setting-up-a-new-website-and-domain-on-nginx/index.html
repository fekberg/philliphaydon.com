Setting up new Website and Domain on nginx

posted on 27 Jun 2013 | Azure
<ul> <li><a href="/2013/06/setting-up-ubuntu-and-nginx-on-azure/">Part 1 - Setting up the Virtual Machine and nginx</a></li> <li>Part 2 - Setting up new Website and Domain on nginx</li> <li><a href="/2013/06/setting-up-mono-on-nginx/">Part 3 - Setting up Mono on nginx</a></li> <li><a href="/2013/07/setting-up-a-nancyfx-website/">Part 4 - Setting up a NancyFX website</a></li> <li><a href="/2013/07/setting-up-a-servicestack-service/">Part 5 - Setting up a ServiceStack web service</a></li> </ul> <p>The purpose of series is to be able to create an end-to-end setup of a Linux Server on Azure and deploy a brand new production NancyFX website or ServiceStack service, and not get confused in-between. Like I was when I sussed all this out. :)</p> <h2>Prelude</h2> <p>When using IIS I think it's safe to say a large majority of us would use the GUI to configure or create new Web Sites. Even though it's actually possible to create a new website and configure it entirely through Powershell.</p> <p>But, as scary as it sounds it's not too hard to configure a new Website in nginx. I actually think its easier than doing it with Apache, despite the fact that they are relatively similar. </p> <!--excerpt--> <h2>Where are sites configured?!?</h2> <p>The nginx configuration files are in <code>/etc</code>. The etc directory is for all system related configuration files. </p> <p>To learn about Linux Filesystem Hierarchy, you can read about them here:</p> <p><a href="http://www.tldp.org/LDP/Linux-Filesystem-Hierarchy/html/">http://www.tldp.org/LDP/Linux-Filesystem-Hierarchy/html/</a></p> <p>Navigate to the nginx directory by running the command <code>cd /etc/nginx</code> once you reach the nginx directory run the command <code>ls</code> to see what's in the directory.</p> <p>The directory listing should look something like.</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-1.png" alt="" /></p> <p>The main things we will be touching in here are <code>sites-available</code> and <code>fastcgi_params</code>, though the <code>fastcgi_params</code> will be in part 3 when we setup Mono. </p> <p><code>fastcgi_params</code> is a configuration file, while the <code>sites-available</code> is a folder. Navigate to the folder by calling <code>cd sites-available</code></p> <p>In here if you call <code>ls</code> you will see there's only 1 file called default. This is the default configuration file, we could add more for nginx but to keep things simple we will simply append our website to this file.</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-2.png" alt="" /></p> <p>Now open the file in nano by calling <code>nano default</code></p> <p><img src="/images/setup-mono-on-ubuntu-part-2-3.png" alt="" /></p> <p>You can see there's a <code>server</code> defined which sort of looks like json. I don't actually know what this formatting is called but it's simply a configuration name with either a value(s) or a scope (defined with braces)</p> <p>The existing server is setup for localhost. We can add our own, so above the server we want to add the following.</p> <pre><code>server { # the port the site is listening on listen 80; # the hostname server_name phillip.pw; # the path to where we host the site root /var/www/phillip.pw; # the default page index index.html; } </code></pre> <p>You should end up with your terminal looking like</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-4.png" alt="" /></p> <p>Now we need to save the file. Do this by pressing <kbd>CTRL</kbd> + <kbd>X</kbd></p> <p>It should then confirm you want to make the changes.</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-5.png" alt="" /></p> <p>Enter <kbd>Y</kbd> to accept the changes, then press <kbd>Enter</kbd> to confirm the filename. Don't change the filename!!!</p> <p>Now you should be back at the terminal. </p> <p><span class="note"><strong>Note:</strong> If it says you don't have permission to change the file then you're not a root user or you didn't run nano as a root user by prefixing the command with <code>sudo</code></span></p> <h2>Creating the site directory</h2> <p>For the configuration we put the root as <code>/var/www/phillip.pw</code>, this path shouldn't exist yet (except for var) Based on my reading of the Linux Filesystem, this seemed like the best place to put it. If someone disagrees or knows where it should go please leave a comment.</p> <p>Navigate to var by calling <code>cd /var</code>, then calling <code>ls</code>, you should see there is no <code>www</code> folder. So lets create one by calling <code>mkdir www</code>, navigate to the directory; <code>cd www</code>. And make the next directory; <code>mkdir phillip.pw</code>.</p> <p>Your console should look something like this</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-6.png" alt="" /></p> <h2>The test file</h2> <p>Now that we are inside our hosting directory we need to create a test file. Call <code>nano index.html</code> which will open up a new nano with nothing in it.</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-7.png" alt="" /></p> <p>Here we can throw some basic HTML in that looks like:</p> <pre><code>&lt;html&gt; &lt;head&gt; &lt;title&gt;Our test page!&lt;/title&gt; &lt;/head&gt; &lt;body&gt; &lt;h1&gt;Hello World from nginx! Our test page works!&lt;/h1&gt; &lt;/body&gt; &lt;/html&gt; </code></pre> <p>Nothing fancy. You can also write this in notepad and then right click on the terminal which will automatically paste the content in.</p> <p>Next hit <kbd>CTRL</kbd> + <kbd>X</kbd> to exit, then <kbd>Y</kbd> followed by <kbd>Enter</kbd> to accept the changes and save the file.</p> <p>Bam we're done! (almost)</p> <h2>Update your domain name!</h2> <p>Lastly we need to update our domain host records. I'm using <www.namecheap.com> for my hosting. Create a new <code>A</code> record for the domain we defined in our site configuration; <code>phillip.pw</code> using the IP from the Azure portal.</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-8.png" alt="" /></p> <p>Be sure to copy the PUBLIC Virtual IP, and not the Internal IP. Otherwise you may get angry when things don't work when they should. :)</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-9.png" alt="" /></p> <p>Save the changes to your domain host records. </p> <p><img src="/images/setup-mono-on-ubuntu-part-2-10.png" alt="" /></p> <p>Running a quick ping test in Windows Command Prompt should resolve to the correct IP.</p> <h2>The BIG Test</h2> <p>So, does it all work?!?</p> <p>Navigating to the website <code>http://phillip.pw</code> yields:</p> <p><img src="/images/setup-mono-on-ubuntu-part-2-11.png" alt="" /></p> <p>Yay! </p> <h2>What else?</h2> <p>So we only configured the domain <code>phillip.pw</code>, but what if we want both that and <code>www.phillip.pw</code>. We can update the servername to <code>*.phillip.pw</code> which will wild-card.</p> <pre><code># the hostname server_name *.phillip.pw; </code></pre> <p>Or if we want to be more specific we can do:</p> <pre><code># the hostname server_name www.phillip.pw phillip.pw; </code></pre> <p>Which will only work for the domain and the <code>www</code> sub domain. </p> <p>That's all there is to it. Next up... Mono! :D</p>
