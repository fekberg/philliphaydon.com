Setting up a NancyFX website

posted on 04 Jul 2013 | Azure
<ul> <li><a href="/2013/06/setting-up-ubuntu-and-nginx-on-azure/">Part 1 - Setting up the Virtual Machine and nginx</a></li> <li><a href="/2013/06/setting-up-a-new-website-and-domain-on-nginx">Part 2 - Setting up new Website and Domain on nginx</a></li> <li><a href="/2013/06/setting-up-mono-on-nginx/">Part 3 - Setting up Mono on nginx</a></li> <li>Part 4 - Setting up a NancyFX website</li> <li><a href="/2013/07/setting-up-a-servicestack-service/">Part 5 - Setting up a ServiceStack web service</a></li> </ul> <p>Time to setup NancyFX on Mono.</p> <h2>Prelude!</h2> <p>This series is done using Mono 2.10 and .NET 4.0. This wont work with a 4.5 project since we need Mono 3.0 for that, but I plan to do another series on building Mono from source since there's no package available yet.</p> <p>Also, this post assumes you've setup FTP to upload the files, I'm not going to go into detail, but you can install <code>vsftpd</code> and Google the setup. If you're new to Linux and followed Parts 1-3 so far, it should be easy enough to setup and install. All you need to do is authenticate using sshftp or sftp, rather than normal ftp. </p> <h2>Creating a Nancy test project</h2> <p>The easiest way to create a test project is to grab the <a href="http://visualstudiogallery.msdn.microsoft.com/f1e29f61-4dff-4b1e-a14b-6bd0d307611a">Nancy Templates</a> from the Visual Studio Gallery. </p> <p>Using this method, we can create a new project in Visual Studio and select Nancy Application.</p> <!--excerpt--> <p><img src="/images/setup-mono-on-ubuntu-part-4-1.png" alt="" /></p> <p>Don't forget to make this a .NET 4.0 project, NOT a .NET 4.5 project. When running Mono 3.0 you can choose .NET 4.5.</p> <p>Once created press F5 and you should end up with a screen like so:</p> <p><img src="/images/setup-mono-on-ubuntu-part-4-2.png" alt="" /></p> <p>BUT, before we can publish, we need to update Nancy to the beta version (version 0.18.0) because 0.17.1 doesn't work on Mono 2.10. Hopefully if you're reading this shortly after its being published 0.18.0 will be out and you can.</p> <hr /> <p>Updating the Nuget Packages. We need to add the Nancy CI builds to out options.</p> <p><img src="/images/setup-mono-on-ubuntu-part-4-4.png" alt="" /></p> <p>And we can remove the references from the project to <code>Nancy</code> and <code>Nancy.Hosting.Aspnet</code></p> <p><img src="/images/setup-mono-on-ubuntu-part-4-5.png" alt="" /></p> <p>And now you can run the commands:</p> <p><code>install-package Nancy</code></p> <p><code>install-package Nancy.Hosting.Aspnet</code></p> <p><img src="/images/setup-mono-on-ubuntu-part-4-6.png" alt="" /></p> <hr /> <p>Now we can publish the website. I'm just going to publish to the file system since this is a learning exercise. </p> <h2>Deployment</h2> <p>Now that we've created and updated our Nancy project, published it. Now we can deploy it! </p> <p><img src="/images/setup-mono-on-ubuntu-part-4-7.png" alt="" /></p> <p>So we can remove the html files we manually created in the previous posts, and deploy our Nancy website to the FTP. </p> <h2>Does it work?</h2> <p>If we visit the site now, most likely we will end up with:</p> <p><img src="/images/setup-mono-on-ubuntu-part-4-8.png" alt="" /></p> <p>This is because we originally configured our fastcgi index to be <code>index.aspx</code></p> <h2>Configure the default page</h2> <p>We can update our nginx configuration now, call <code>nano /etc/nginx/sites-available/default</code></p> <p><img src="/images/setup-mono-on-ubuntu-part-4-9.png" alt="" /></p> <p>Remove the <code>index.aspx</code> part so the index is just <code>/</code> and nothing else.</p> <p>Now we can restart nginx to make sure the changes are picked up... <code>service nginx restart</code></p> <h2>Now does it work?!?</h2> <p>Now we can give it a restart anddddddddddd.....</p> <p><img src="/images/setup-mono-on-ubuntu-part-4-10.png" alt="" /></p> <p>BAM! Now we have Nancy... Running on Mono... on nginx... Ubuntu... on Azure...</p> <p>How awesome is that! </p>
